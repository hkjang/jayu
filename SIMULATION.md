## 🏗️ 아키텍처 개요

```
┌─────────────────────────────────────────────────────────┐
│                   시뮬레이션 실행 흐름                    │
└─────────────────────────────────────────────────────────┘

yfinance → 2년치 OHLCV 데이터
     ↓
기술 지표 계산 (RSI, EMA10/20/50, 거래량비율, GAP, ATR)
     ↓
┌──────────────────────┐
│   파라미터 생성       │  ← 유전자 풀(60%) + 랜덤(40%)
│  - 유전: 변이/교차    │
│  - 랜덤: 공간 탐색    │
└──────────────────────┘
     ↓
Walk-Forward 백테스트
  ├── 학습셋 (18개월): 진입/청산 시뮬레이션
  └── 검증셋 (6개월): 과최적화 방지
     ↓
성과 평가 (승률, Sharpe, PF, 수익률, MDD)
     ↓
합격 기준 통과?
  ├── Yes → 유전자 풀 업데이트 + best_strategy.json 갱신
  └── No  → 다음 시도
     ↓
메타 학습 업데이트 (파라미터별 성공률 기록)
     ↓
카카오톡 결과 전송
```

## 🧬 유전 알고리즘

### 핵심 아이디어
매 실행 시 단순히 무작위 탐색하지 않고, **이전에 잘 작동한 전략을 변이/교차**하여 탐색 효율을 높임.

### 유전자 풀 (gene_pool.json)
- 종목별 상위 10개 전략 보관
- Sharpe ratio 기준 정렬
- 매 실행마다 새 최적 전략으로 갱신

### 선택 방법
```
60%: 유전 탐색
  └── 40%: 두 부모 교차 (crossover)
  └── 60%: 한 부모 변이 (mutate, 변이율 30%)

40%: 완전 랜덤 (새 영역 탐색)
```

### 변이 (Mutation)
각 파라미터를 30% 확률로 독립적으로 변경.
RSI, EMA, 거래량, 손절/목표, 보유일 전체 대상.

### 교차 (Crossover)
두 부모 전략에서 각 파라미터를 50% 확률로 선택.

## 📐 Walk-Forward 검증

### 목적
과최적화(Overfitting) 방지 — 학습 데이터에서만 좋은 전략 제거.

### 구조
```
전체 2년 데이터
├── 학습셋: 앞 18개월 (백테스트로 파라미터 최적화)
└── 검증셋: 최근 6개월 (실제 성과 검증)
```

### 합격 기준
1. 학습셋: 승률 ≥ 48%, PF ≥ 1.2, 수익률 > 0
2. **검증셋 수익률 > 0** (핵심 — 미래 데이터에서도 수익)
3. 검증셋 Sharpe ratio 기준으로 최종 순위 결정

---

## 📊 성과 지표

| 지표 | 설명 | 합격 기준 |
|------|------|-----------|
| 승률 (Win Rate) | 수익 거래 / 전체 거래 | ≥ 48% |
| 수익 팩터 (PF) | 총수익 / 총손실 | ≥ 1.2 |
| 샤프 비율 (Sharpe) | 위험 조정 수익률 | 최대화 |
| 최대 낙폭 (MDD) | 최고점 대비 최대 손실 | 참고용 |
| R:R 비율 | 평균 수익 / 평균 손실 | > 1.5 |

---

## 🎛️ 파라미터 공간

| 파라미터 | 선택지 | 의미 |
|----------|--------|------|
| rsi_lo | 35~55 | RSI 하단 진입 기준 |
| rsi_hi | 60~80 | RSI 상단 진입 기준 |
| ema_span | 10, 20, 50 | 추세 확인용 EMA |
| vol_mult | 1.2~3.0 | 거래량 급증 배수 |
| gap_min | -1%~+0.5% | 최소 갭 조건 |
| stop_pct | 2~5% | 손절 비율 |
| target_pct | 4~15% | 목표 수익 비율 |
| hold_days | 1~5일 | 최대 보유일 |
| pos_size | 15~30% | 포지션 비중 |

**제약 조건**: `target_pct ≥ stop_pct × 1.5` (최소 R:R 1.5:1 보장)

## 🧠 메타 학습 (meta_learning.json)

각 파라미터 값의 **역대 성공률**을 추적.

예시:
```json
{
  "ema_span": {
    "50": {"wins": 123, "total": 200},  ← EMA50 성공률 61.5%
    "20": {"wins": 89,  "total": 180},  ← EMA20 성공률 49.4%
    "10": {"wins": 67,  "total": 160}   ← EMA10 성공률 41.9%
  }
}
```

향후 활용: 성공률 높은 파라미터에 더 높은 샘플링 확률 부여.

## 📈 진입/청산 로직

### 진입 조건 (4가지 동시 충족)
```python
1. rsi_lo ≤ RSI ≤ rsi_hi          # 과매수/과매도 아닌 구간
2. 종가 > EMA(ema_span)            # 추세 상승 중
3. 거래량비율 ≥ vol_mult           # 거래 활성화
4. 갭 ≥ gap_min                    # 갭다운 필터
```

### 청산 조건 (우선순위 순)
```
1. 손절: 저가가 (진입가 × (1 - stop_pct)) 이하 → 즉시 손절가 청산
2. 목표: 고가가 (진입가 × (1 + target_pct)) 이상 → 즉시 목표가 청산
3. 시간: hold_days 경과 → 종가 청산
```

### 포지션 사이징
```
투자 금액 = 보유 자본 × pos_size (15~30%)
→ 고정 비율 베팅 (Kelly Criterion 단순화)
```

## 🔄 실행 주기 및 자율 개선 흐름

```
[매 4시간]
실행 1회 (500회×6종목 = 3,000 시뮬레이션)
     ↓
하루 6회 × 3,000 = 18,000회/일
     ↓
매 실행마다 유전자 풀 개선
     ↓
1주일 후: 42회 실행, 126,000회 누적
→ 파라미터 탐색 공간 거의 전체 커버
     ↓
1개월 후: 유전자 풀이 수렴 → 안정적 최적 전략 도출
```
